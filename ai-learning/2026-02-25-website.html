🚀 用 Kimi + OpenClaw 搭建个人网站

零代码基础，全程对话完成网站搭建和部署

前言

今天（2026年2月25日）我完成了一个"不可能的任务"——零代码基础搭建个人网站。

整个过程只需要和 Kimi AI 助手对话，所有技术细节都由 AI 处理。从购买域名到网站上线，全程不到 3 小时。

使用的工具

🤖 Kimi AI 助手

月之暗面开发的 AI 助手，负责：

  • 生成网站 HTML/CSS/JS 代码
  • 配置 Cloudflare Pages 部署
  • 编写自动化脚本
  • 解决各种技术问题

⚡ OpenClaw

自动化任务调度工具，负责：

  • 定时检测新文章
  • 自动推送到 GitHub
  • 触发 Cloudflare 重新部署

☁️ Cloudflare Pages

免费网站托管平台，提供：

  • 全球 CDN 加速
  • 自动 HTTPS 证书
  • GitHub 集成部署
  • 免费自定义域名

🐙 GitHub

代码托管平台，作为：

  • 网站代码仓库
  • 版本控制
  • Cloudflare 部署源

搭建过程

第一步：明确需求

我告诉 Kimi：

"我想搭建一个个人博客网站，主要写供应链相关的文章。要求：

  • 全自动发布（我只需要写文章）
  • SEO 优化（让搜索引擎能搜到）
  • 免费托管（不想买服务器）
  • 自定义域名

我没有任何代码基础，全靠你帮我。"

第二步：选择技术方案

Kimi 推荐了以下方案：

  • 静态网站生成器 Hugo - 速度快、SEO 友好
  • GitHub 仓库 - 免费代码托管
  • Cloudflare Pages - 免费部署和 CDN
  • OpenClaw - 自动化发布流程

第三步：配置环境

在 Kimi 的指导下，我完成了：

  1. 注册 GitHub 账号
  2. 创建代码仓库
  3. 注册 Cloudflare 账号
  4. 配置 Pages 项目
  5. 设置 API Token

全程只需要复制粘贴 Kimi 提供的代码和链接，不需要理解技术细节。

第四步：生成网站代码

Kimi 帮我生成了完整的网站代码：

  • 首页（index.html）
  • 文章页面模板
  • 关于页面
  • 响应式 CSS 样式
  • Schema.org SEO 标记

第五步：配置自动化

使用 OpenClaw 配置定时任务：

每小时检测：检查是否有新文章

自动润色：AI 优化文章标题、摘要

自动生成 SEO 数据：Schema.org JSON-LD

自动推送：推送到 GitHub

自动部署：Cloudflare Pages 重新构建

遇到的挑战

挑战1：GitHub 推送失败

问题：网络连接超时，代码推不上去。

解决：Kimi 改用 Cloudflare API 直接部署，绕过 Git 推送。

挑战2：网站显示空白

问题：部署成功但页面空白。

解决：发现是 Hugo 主题依赖问题，Kimi 将网站改为纯 HTML，无需构建。

挑战3：供应链专栏文章格式

问题：Markdown 文件在网站上显示为纯文本。

解决：Kimi 将 Markdown 转换为完整的 HTML 页面，包含样式和导航。

  • ✅ 供应链前沿日报专栏（每日自动更新）
  • ✅ AI学习应用专栏（本文所在）
  • ✅ 关于页面
  • ✅ 完整的 SEO 优化

心得体会

1. AI 降低了技术门槛

以前搭建网站需要学习 HTML、CSS、JavaScript、Git、服务器配置等一大堆技术。现在只需要会打字，把需求告诉 AI，它就能帮你完成。

2. 对话式开发更高效

不需要看教程、查文档，有问题直接问 AI。它比搜索引擎更精准，比教程更个性化。

3. 自动化是核心

配置好自动化流程后，我只需要专注于写文章，发布、部署、SEO 都交给系统自动完成。这才是"个人网站"应有的体验。

4. 免费方案足够用

GitHub + Cloudflare Pages + OpenClaw 的组合完全免费，性能却非常好（全球 CDN 加速）。对于个人博客来说，不需要花钱买服务器。
